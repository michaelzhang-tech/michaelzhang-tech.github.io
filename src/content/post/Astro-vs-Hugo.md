## What I Observed
Hug is fast and easy to build up websites, and there are multiple pre-built themes to choose.
While, that is true before you push to github pages. All kinds of issues happened and really hard to fix, especially
when the themes are black boxes.

One the other hand, you need more design and manual creation of directory, files, content when using Astro, but I can build up my understanding about how Astro works through this way.
More importantly, github pages works much much better with Astro and I got a pretty good start compare to hugo.

## Why It Matters

- Themes are not that important, conent is!
- Build from scratch seems slower, but it is easy to fix things since you know what you have done
- Content-driven is great for personal knowledge website or any website

## Key Insight

Slower is faster, faster is slower! At least in this case.

## Closing Thought
Try Astro + github pages, it works great!
The minimal starter is already good enough and I can focus on content but not website building and troubleshooting.
